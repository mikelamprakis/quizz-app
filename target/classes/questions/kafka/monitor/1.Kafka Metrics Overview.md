```markdown
Which Kafka metric indicates the time it takes for a record to be acknowledged by all in-sync replicas?
```

**Options**

```markdown
- A. ReplicationLag
- B. FetchLatency
- C. ProducerLatency
- D. ISRTime
```

<details><summary>Response:</summary>

**Answer:** A

**Explanation:**

```markdown
`ReplicationLag` tracks the delay between leader and replica acknowledgment.

- A. Correct: Measures lag in syncing replicas.
- B. Related to how fast consumers fetch—not replication.
- C. Related to producer delay, not replication.
- D. Not a standard Kafka metric.
```

</details>

## Question 44

```markdown
Why is it important to monitor the `RequestRate` metric in Kafka?
```

**Options**

```markdown
- A. To measure the number of messages being produced
- B. To measure the number of bytes being consumed
- C. To measure the rate of requests being handled by Kafka brokers
- D. To measure the number of partitions
```

<details><summary>Response:</summary>

**Answer:** C

**Explanation:**

```markdown
`RequestRate` helps assess Kafka broker load, which impacts performance under pressure.

- A. Related to `MessagesInPerSec`.
- B. Measured by `BytesInPerSec` or `BytesOutPerSec`.
- C. Correct: It’s about how many requests brokers handle.
- D. Not related to runtime performance.
```

</details>
